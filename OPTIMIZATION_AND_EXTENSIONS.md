# MDNote 优化与拓展建议

> 基于当前仓库技术栈与已实现能力（侧栏树形结构、`@dnd-kit` 笔记归档、本地存储、Markdown 工作区、导入与打印等）整理，便于分阶段迭代。

---

## 一、侧栏与拖拽（DND）

| 方向 | 建议 | 说明 |
|------|------|------|
| **可发现性** | 在首次进入笔记面板或空状态文案中提示「可拖入文件夹」 | 降低学习成本，避免用户只依赖「移动到文件夹」菜单。 |
| **键盘与无障碍** | 为拖拽补充键盘路径，或提供等效的「移动到…」快捷入口 | 满足 WCAG；当前以指针为主，屏幕阅读器用户依赖菜单更合理。 |
| **拖拽预览** | 使用 `DragOverlay` 显示浮动缩略卡片（标题 + 图标） | 长列表滚动时，原行半透明可能不够醒目，浮层反馈更清晰。 |
| **根目录投放** | 增加显式「根目录」投放区（或列表顶部条） | 与菜单中「根目录」行为对齐，减少「只能拖到文件夹」的心智负担。 |
| **冲突策略** | 长距离拖拽时考虑 `autoScroll` 或自定义滚动容器加速 | 树很深时，拖到底部文件夹更省力。 |
| **拓展：排序** | 引入 `@dnd-kit/sortable` 维护笔记/文件夹顺序字段 | 需扩展数据模型（如 `sortIndex`）与持久化，与「仅归档」策略解耦。 |
| **拓展：多选** | Shift/Ctrl 多选 + 批量拖入文件夹 | 适合笔记量大的场景，需状态机与冲突处理。 |

---

## 二、数据与持久化

| 方向 | 建议 | 说明 |
|------|------|------|
| **容量与性能** | 监控 LocalStorage 体积，大正文时做惰性加载或 IndexedDB 迁移 | 单域 5MB 量级限制，长文多笔记时易触顶。 |
| **版本与迁移** | 为存储结构增加 `schemaVersion`，集中做迁移逻辑 | 避免改字段名后老用户数据静默损坏。 |
| **导入冲突** | 可选「覆盖 / 跳过 / 重命名」策略 UI | 当前重名策略已文档化，交给用户选择会更可控。 |
| **拓展：同步** | 可选后端或云盘（WebDAV / S3 / 自建 API） | 单机 LocalStorage 适合原型；长期需多端一致。 |
| **拓展：导出** | 一键导出为 ZIP（保留文件夹路径） | 与「文件夹导入」形成闭环，便于备份与迁移。 |

---

## 三、编辑器与工作区

| 方向 | 建议 | 说明 |
|------|------|------|
| **编辑体验** | 大纲（Heading 树）、当前标题在侧栏或迷你 TOC 中高亮 | Markdown 长文导航刚需。 |
| **搜索** | 搜索结果内高亮匹配片段、跳转到正文位置 | 当前跨文件夹检索可再加强「落地」体验。 |
| **快捷键** | 统一快捷键表（新建、保存语义、切换预览、聚焦搜索） | IDE 风格用户期望高；可配合 `cmd+k` 命令面板。 |
| **拓展：分屏** | 双栏对照编辑或「笔记 A 引用笔记 B」 | 知识库类场景常见，复杂度高。 |

---

## 四、Markdown 与安全

| 方向 | 建议 | 说明 |
|------|------|------|
| **XSS 面** | 对 `rehype-raw` 评估白名单标签/属性，或默认关闭 raw、高级用户再开启 | 项目文档已提示 XSS 风险，产品化时应收敛默认行为。 |
| **外链** | `rel="noopener noreferrer"`、可选 `nofollow` | 预览区链接安全与 SEO 习惯。 |
| **拓展：Mermaid / KaTeX** | 按需插件化，避免首包膨胀 | 与 Vite 动态 import 结合较自然。 |

---

## 五、性能与工程化

| 方向 | 建议 | 说明 |
|------|------|------|
| **分包** | 对 Prism 语言包、可选 Markdown 插件做 `import()` | 缓解构建告警中的大 chunk 问题。 |
| **列表渲染** | 笔记极多时对树虚拟化（如 `@tanstack/react-virtual`） | 避免长列表 DOM 与重排成本。 |
| **测试** | 为核心 reducer/移动笔记逻辑补充单元测试；DND 关键路径用 RTL 冒烟 | 拖拽依赖浏览器行为，单测优先纯逻辑。 |
| **CI** | `lint` + `tsc -b` + `build` 在 PR 上必跑 | 低成本防回归。 |

---

## 六、UI / 品牌与部署

| 方向 | 建议 | 说明 |
|------|------|------|
| **响应式** | 窄屏下侧栏折叠策略、触摸友好的拖拽阈值 | 平板/小屏上 5px 激活阈值可单独调优。 |
| **主题** | 可选浅色主题或跟随系统 `prefers-color-scheme` | 扩大受众；需梳理 `prose` 与侧栏 token。 |
| **部署** | Vercel 环境变量、缓存策略、`/assets` 与路由 history 模式说明 | 与 `PROJECT_TECH_STATUS.md` 中部署意图对齐。 |

---

## 七、建议优先级（主观）

1. **高**：存储 schema 版本与迁移；`rehype-raw` 风险收敛或文档化「高级选项」；拖拽可发现性 + 可选 `DragOverlay`。  
2. **中**：根目录投放区、大纲/TOC、搜索结果高亮、CI 流水线。  
3. **低**：多选批量拖拽、云同步、ZIP 导出、分屏编辑。

---
